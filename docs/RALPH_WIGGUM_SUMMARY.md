## ğŸ“ METHODOLOGY REFERENCE

### Ralph Wiggum Technique

**Source:** Wall Street Journal (2026-01-09)  
"The Ralph Wiggum Technique: How AI is Transforming Software Debugging"

**Core Concept:** Use AI assistants to iterate rapidly through observe-diagnose-test loops, treating code like a doctor treats symptoms:

1. Look at what's broken
2. Find the root cause
3. Propose a fix
4. Test it
5. Refine based on results
6. Repeat until fixed

**Why It Works:**

- AI can process vast amounts of code quickly
- Humans verify safety of each change
- Reversible steps (roll back if needed)
- Fast iteration cycles
- Learns from each attempt
